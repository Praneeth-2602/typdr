Project Concept — SwiftKeys
A typing platform that
actually teaches you.
Serverless, local-first. From finger placement for beginners to per-key analytics for speed freaks — with no backend to maintain.
What monkeytype gets wrong
01
No structured learning path
Assumes you can already touch type. No finger placement, no lessons, no hand overlay. Beginners are dropped in the deep end.
02
Inflated WPM scores
Default 200-word list with no punctuation. Your "90 WPM" in Monkeytype might be 60 WPM in a real email.
03
No per-finger analytics
You get total WPM and accuracy — but no breakdown of which fingers, keys, or bigrams are actually slowing you down.
04
No developer code mode
Random "programming words" isn't IDE practice. Developers need real structured code with indentation, brackets, and syntax.
05
Reinforces bad habits
Repeated speed tests don't fix bad technique. Hunt-and-peck users can game WPM without ever improving their form.
06
No guided improvement
No AI-driven weak spot detection, no custom drill generation. You're on your own to figure out why you're plateauing.
